# Accessibility Audit

Review codebase for accessibility issues and ensure WCAG compliance.

## Steps

1. **Semantic HTML review**
   - Check for proper heading hierarchy
   - Verify form labels and associations
   - Ensure proper use of semantic elements

2. **ARIA and keyboard navigation**
   - Review ARIA attributes
   - Test keyboard navigation
   - Verify focus management

3. **Visual and interaction**
   - Check color contrast ratios
   - Verify focus indicators
   - Test with screen readers

## Accessibility Checklist
- [ ] Semantic HTML used throughout
- [ ] Proper heading hierarchy
- [ ] Form labels associated
- [ ] Keyboard navigation works
- [ ] ARIA attributes where needed
- [ ] Color contrast meets WCAG AA
- [ ] Focus indicators visible
