# Add PR Description

Generate or enhance a pull request description with context, changes, and testing information.

## Steps

1. **Analyze changes**
   - Review git diff
   - Identify key changes
   - Note breaking changes

2. **Write description**
   - Summarize changes clearly
   - Explain motivation and context
   - List testing performed
   - Note any breaking changes

3. **Format and structure**
   - Use clear headings
   - Include checkboxes for reviewers
   - Link related issues
   - Add screenshots if applicable
