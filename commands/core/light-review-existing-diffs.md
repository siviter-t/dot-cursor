# Light Review Existing Diffs

Perform a quick review of existing git diffs to catch obvious issues before deeper review.

## Steps

1. **Identify changes**
   - Review git diff output
   - Focus on changed files
   - Note file additions/deletions

2. **Quick checks**
   - Look for obvious bugs
   - Check for removed functionality
   - Verify no accidental commits

3. **Flag concerns**
   - Mark areas needing deeper review
   - Note potential issues
   - Suggest improvements
