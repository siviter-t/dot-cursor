# Write Unit Tests

Create comprehensive unit tests for code changes, ensuring good coverage and test quality.

## Steps

1. **Identify test targets**
   - Review changed code
   - Identify functions and components to test
   - Determine edge cases

2. **Write tests**
   - Create test files following project structure
   - Write tests for happy paths
   - Add tests for edge cases and error handling

3. **Verify coverage**
   - Run test suite
   - Check coverage metrics
   - Ensure tests are maintainable

## Test Quality Checklist
- [ ] Tests cover main functionality
- [ ] Edge cases tested
- [ ] Error handling tested
- [ ] Tests are readable and maintainable
- [ ] No flaky tests
